## 2. Phase 3C 価格推移グラフのレビュー

### 2.1 機能評価

**DS**: 「recharts の LineChart + 駅ラインと路線平均の2系列、シンプルで良い設計。XAxisが古い→新しいの左→右になってる。前回X軸が逆だって直したやつだよね」

**Dev**: 「YEARS定数が新しい順で定義されてるのをreverseしてる。地味なバグだったけど、直してよかった」

**PdM**: 「"最近10年 / 全期間"のトグルもある。デフォルト10年表示は正解。2005年のデータを全部表示してもユーザーの意思決定には関係ないから」

**Sales**: 「取引件数タブが地味にいい。川口駅を見たら件数グラフが2021〜2022年にピークがあった。コロナ明けの需要増加が見えてて、これをお客さんに説明する材料になる」

**DS**: 「ただ路線平均と比較するとき、"どの路線か"を選べるようになってる。これ、複数路線にまたがる駅（例: 大宮）だとどうなる？」

**Dev**: 「selectedLineで一路線だけ選べる形。デフォルトは1番目の路線。複数路線駅は最初のやつが出る。明示的に"別の路線と比較"ボタンで切り替えられる」

**DS**: 「実用上は問題ないと思うけど、大宮みたいな巨大ターミナルは複数路線の影響があるから、そこだけ注意が要るかな。まあデプロイ後に実際のFBで判断すれば十分」

### 2.2 年度範囲フィルタとの連動

**PdM**: 「年度フィルタがyearFrom〜yearToになったのはグラフとの相性がすごく良くなった。"2020〜2025年データでマップ表示"と"その期間の価格推移グラフ"が同じ文脈でつながってる」

**DS**: 「加重平均マージのロジック、`mergeStatsList()`でやってる。年数を増やすと取引件数が増えてデータの信頼性が上がる一方、古い価格が混ざって現在の相場からズレる。ユーザーがそのトレードオフを意識できるかどうかが課題」

**PdM**: 「フィルタパネルに"2020〜2025年"って表示されてればユーザーは理解できると思う。年度範囲を広げると"過去データも含む"ことは自明だから」

**PdM**: 「それはB-10（免責事項文言）に含めましょう。C-01のアクションアイテムとして」

**Dev**: 「了解」

**Sales**: 「もう一点。"憧れエリア提案"で出てくる類似駅、AIの理由文が面白い。"同じ埼玉高速鉄道沿線で生活利便性が類似している"みたいな。ただ、類似駅の選択根拠が"同一路線+30点"というのが見えてないユーザーには、なぜその駅が出てきたのかわかりにくいかも」

**DS**: 「スコアが高い駅だから出てるわけだけど、ユーザーには"路線が近い"とか"エリアが隣接"みたいな理由を明示的に見せるほうがいいかもね」
