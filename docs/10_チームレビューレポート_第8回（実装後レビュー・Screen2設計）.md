## 5. バグ報告: AIエリア提案の価格計算（修正済み）

**Dev**: 「今回セッションで修正したバグを共有する。`/api/suggest/route.ts` の `matchStations` 関数が `const area = conditions.targetArea ?? 70` で固定していたため、UIで80㎡を設定していてもAI提案の価格は常に70㎡換算で計算されていた」

**Sales**: 「それ、80㎡設定で提案を実行したら入間市駅が994万と出てきたやつですね。70㎡換算だと確かに低く見える」

**Dev**: 「修正内容は3点: ① `SuggestPanel.tsx` の fetch ボディに `targetArea` を追加、② `page.tsx` から `SuggestPanel` に `targetArea={filter.targetArea}` を渡す、③ `route.ts` でリクエストボディの `targetArea` を受け取ってクエリに面積指定がない場合はUIの値を使う」

**DS**: 「ところで、SuggestPanel の localStorage キャッシュのキーが `"suggest_cache_" + query` だけになってる。これ、同じクエリでも `targetArea` が違えば結果が変わるはずなのに、キャッシュキーに `targetArea` が含まれていない。古い70㎡ベースのキャッシュが残ってると、80㎡設定で検索しても古い結果が返ってくることになる」

**Dev**: 「鋭い指摘。`CACHE_KEY_PREFIX + query + "_" + targetArea` のようにキーに `targetArea` を含めるべきだな。これは H-02 として対応しよう」

**PdM**: 「もう一個。OnboardingWizard の STEP 3 で `/api/suggest` を呼び出すとき、`targetArea` をリクエストボディに含めていないのでは？」

**Dev**: 「確認すると... ウィザードの `handleComplete` 関数は `body: JSON.stringify({ query })` で `targetArea` を渡していない。ウィザードで 30㎡ を選んでも、STEP 3 の AI 提案は 70㎡ 換算になってしまう。SuggestPanel は直したけど、ウィザード側がまだ。H-03 として修正する」
