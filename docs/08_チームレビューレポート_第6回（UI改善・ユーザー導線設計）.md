# チームレビューレポート 第6回: 埼玉県 中古マンション相場マップ

**日時**: 2026-03-16
**参加者**: 開発エンジニア（Dev）、デザイナー出身PdM（PdM）、データサイエンティスト（DS）、不動産セールス（Sales）
**形式**: UX導線レビュー + UI改善設計議論
**議題**: 主要機能（予算シミュレーター・AIエリア提案・他県比較）がヘッダー右上に並列配置されていて、ユーザーが自然な探索フローを辿れていない問題の解消

**PdM**: 「そう。今回オーナーから具体的な課題が出ていて、"ヘッダー右上に機能ボタンが並んでるだけで、ユーザーが直感的に使い方を理解できない" という指摘。スクリーンショットを見るとよくわかる。緑の予算ボタン・青のAIボタン・紫の他県ボタンが横並びで、全部同じ見た目の丸ボタン。優先順位も使う順序もわからない」

## 2. 現状UIの問題解剖

**PdM**: 「現在の構造を整理すると、ヘッダーに4つの要素が右側に並んでる。①他県比較ボタン（紫）②予算シミュレーターボタン（緑）③AIエリア提案ボタン（青）④データ鮮度バッジ（黄）。スクリーンショットで見ると、並び順が "他県比較→予算→AI" の順で、想定ユーザージャーニーとは逆順になってる」

**Dev**: 「page.tsxの83〜113行目を見ると、ボタンはすべて`inline-flex rounded-full px-2.5 py-1 text-xs`で統一されたスタイル。視覚的な重みが全部同じ。PCでは文字が見えてるけど、SPだと "他県" "予算" "AI提案" の3文字ラベルになる」

**Sales**: 「うちのお客さんに見せたら、まず地図を眺めるんですよね。ボタンに気づかないか、気づいても "後で見よう" って後回しにする。まず何をすればいいかが全然伝わってないと思う」

**DS**: 「ユーザーの認知負荷の観点から言うと、問題は3つある。①機能の優先順位がわからない ②使う順序がわからない ③機能を使ったあとの "次のアクション" へのブリッジがない。予算を設定した後にどうすればいいかが全くガイドされてない」

**PdM**: 「想定してた理想のフローをはっきり言うと、"予算シミュレーター → マップで気になるエリアをクリック → AI提案や他県比較で候補を広げる" という3ステップ。でも今のUIに入れると全部横並びのボタンになるので、ジャーニーの感覚がゼロ」

**Dev**: 「コード的に言うと、`budgetOpen / suggestOpen / crossPrefOpen` の3つのstate管理はきれいに分離されてる。問題はUIの "入口" の設計だけ。ロジックは変える必要がない」

## 3. 解決策の選択肢を洗い出す

**PdM**: 「課題がわかったので解決策の選択肢を出していこう」

**Dev**: 「選択肢A: ヘッダーボタンに番号（①②③）を付けて順序を示す。実装コスト最小。ただ番号をつけるだけで "何をするのか" の説明はない」

**Sales**: 「番号だけじゃお客さんに伝わらないと思う。"まず予算を確認してください" くらいのコピーが欲しい」

**DS**: 「選択肢B: 予算未設定のとき、マップのオーバーレイやToastで "まず予算を設定してみましょう" を表示する。動線の起点をプッシュする」

**PdM**: 「選択肢C: ヘッダーのボタン配置をユーザージャーニー順に変えて、かつ予算シミュレーターだけ大きいCTA（Call to Action）ボタンにする。他の2つはサポート機能扱いでサブのサイズに落とす」

**Dev**: 「選択肢D: ヘッダーは最小限にして、フィルターパネルかマップの下に "ガイドバー" を置く。"STEP 1: 予算確認 → STEP 2: エリア探索 → STEP 3: 候補を絞る" みたいな横並びのステップ表示」

**Sales**: 「Dが直感的でいいと思う。お客さんに不動産ポータルで検索するとき、"まず条件を設定してください" という画面があると自然に使えますよね。あれと同じ感覚」

**PdM**: 「いくつか組み合わせる余地もある。ただし "一番効果が高くて実装コストが低いもの" から始めよう。完璧なオンボーディングを最初から作るより、まずは導線を整理する」

## 4. 改善方針の決定

**PdM**: 「議論を踏まえて、改善方針を3つに絞りたい」

**Dev**: 「1つ目: ヘッダーの視覚的階層を変える。予算シミュレーターを "最初にやること" として強調 → ボタンを大きくするかラベルを工夫する。AI提案・他県比較は "任意のサポート機能" として小さく横並び」

**PdM**: 「2つ目: 予算未設定のとき、マップエリアのどこかに "まず予算を確認してみましょう →" という控えめなコールアウトを表示する。AreaListの上部あたりが邪魔にならないスペース」

**DS**: 「3つ目: 予算設定後のフィードバックを強化する。budgetMaxが設定されたら、フィルターパネルに "予算: X,XXX万円 ✕" というバッジが表示されて、マップのグレーアウトが視覚的に効いてるのがわかるようにする」

**Sales**: 「これだけでだいぶ変わると思う。今は予算設定してもどこにも反映されたように見えないから。"入力 → 反映 → 確認" のサイクルが完結してない」

**Dev**: 「E-04（予算外グレーアウト）の話と繋がるな。マップマーカーが変わるだけでなく、フィルターエリアにもバッジを出すと連動感が出る」

**PdM**: 「もう一点。ヘッダーのボタン並び順を変えることも重要。今は "他県比較→予算→AI" の順だけど、"予算→AI提案→他県比較" のジャーニー順に変える。コード1行の変更だけど認知的に大事」

**Dev**: 「page.tsxの83〜113行目でボタンの並び順を変えるだけ。3行のDOM順変更で済む」

## 5. ステップバーの設計議論

**PdM**: 「ステップバーのアイデアをもう少し掘り下げたい。フィルターパネルとマップの間、あるいはAreaListの上部に、3ステップのガイドを置く案はどう？」

**Dev**: 「実装的には新規コンポーネント `JourneyBar` を作る形。表示条件は "budgetMax === null のとき" に限定して、予算設定後は自動で消える。高さ40px以内に収めればマップのスペースを圧迫しない」

**DS**: 「3ステップを全部表示するより、今ユーザーがどのステップにいるかを示すプログレス型がいいと思う。budgetMax未設定 → STEP1ハイライト、budgetMax設定済み → STEP2ハイライト、highlightedStations存在 → STEP3ハイライト」

**Sales**: 「そこまで細かくしなくても、最初にSTEP1が光って、予算設定したら消える、くらいのシンプルさでいい気がする。難しく考えすぎると実装に時間かかるし」

**PdM**: 「Salesの言うとおり。最初は "予算設定促進" だけでいい。全体的なジャーニーガイドはその次のフェーズに回そう。今回は "まず予算から始める" を伝えることに絞る」

**Dev**: 「それなら追加コンポーネント不要で行けるかも。AreaListの上部か、マップの上に重なるような形でbadge/bannerを置くだけ。 `filter.budgetMax === null` のときのみ表示して、`setBudgetOpen(true)` を呼ぶリンクにする」

## 6. 不動産ドメイン視点からの追加指摘

**Sales**: 「UIの話から少し離れて、ユーザーが実際に "何を知りたいか" という観点でも言いたいことがある。うちのお客さんが最初に聞いてくること、1位は "この予算でどこに住めますか？" なんですよ。2位が "どの駅が相場安いですか？" 3位が "新しい物件と古い物件で予算どのくらい違いますか？"」

**PdM**: 「このプロダクトはまさにその1〜2位に答えるツールなんだよね。でも今のUIだと1位の回答が "ヘッダーの小さいボタンをクリック" という難しさで隠れてる」

**Sales**: 「そう。お客さんが直感的に使えるとしたら、最初のページに "あなたの年収はいくらですか？" という入力があって、すぐ地図が変わる体験が一番強い。でもそこまでやると大改修になりますよね」

**Dev**: 「そこまでのフルリデザインはスコープ外として、今できる範囲で "予算を起点にした導線" を強化するのが現実的かな」

**DS**: 「ちなみに今のBudgetPanelは年収とか資産のボタン選択式で、操作感はよい。問題は "入口" が小さいこと。中身は変えなくていい」
